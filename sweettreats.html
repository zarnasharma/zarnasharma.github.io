Introduction

Sweet Treats

Summary

The brief given - Create a fictional restaurant website; should have a rigged contest that the customer never wins. No plugin may be used for this feature; needs to be cross-browser compatible. Besides jQuery, no external libraries are allowed (no Bootstrap or Foundation etc.). This was a team project, I was responsible for the Home Page, and the Contest Page.

Technologies

HTML5, CSS3, Vanilla JavaScript

The Process

Project Sweet Treats - Process

1

Research

For the home page, I was required to develop an image carousel, a daily specials box, and a categories grid. For the contest page, the requirement expected the customer to never win. My research mostly comprised of finding out different types of restaurant contests, and how to not give away that it was rigged.

2

Storyboarding

I found out that most contests were lucky-draw based, and didn't really require much user interaction. In order to build something engaging and interesting, after my team's approval, I decided to develop a simple, rigged memory game around the concept of Cookie Monster as ours was a desserts-only restaurant.

3

Phase Planning

I decided to build the home page first, as it was a common page that linked to all other pages developed by my other teammates. For the contest page, I planned to build a game that would encourage the user to collect a set amount of cookies in order to win a special gift from the restaurant. However, apart from giving a subtle hint about the Cookie Monster being on the loose, the user would not know that the game terminates at any level before the target.

4

Development

I chose to build all my features using Vanilla JavaScript, and not jQuery. The memory game has 3 fallback points, mostly to ensure that the user does not guess that the game is rigged, at least not in the first couple of trials: the game terminates automatically at level 11, there is a random number function that runs during the gameplay, and the moment the expected number is generated, the game terminates, and finally, there is a timer that ends the game after 30 seconds, if the user gets stuck at a level, or does not respond. Responsiveness of the website was taken care by the simple grid system I use in all my projects. You can check it out here.

5

Tests & Tweaks

The interesting part about this phase was when I carried out user tests and found out that it was, indeed, difficult for the users to gauge that the game was rigged.
